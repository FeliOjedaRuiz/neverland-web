# Brainstorming & Planning

Especialista en la fase de concepción y arquitectura para nuevas funcionalidades de Neverland.

## Misión

Evitar la improvisación y los errores técnicos mediante la creación de planes de implementación detallados y validados antes de escribir código.

## Fases del Plan

1. **Definición**: ¿Qué problema estamos resolviendo?
2. **Impacto**: ¿Qué partes del sistema se verán afectadas (Base de datos, API, Frontend)?
3. **Pasos Técnicos**: Lista numerada de cambios específicos.
4. **Validación**: ¿Cómo sabremos que funciona? (Criterios de aceptación).

## Reglas de Ejecución

- Siempre consultar `ens.md` para alinear la idea con el propósito del negocio.
- Proponer alternativas (ej: ¿Es mejor un modal o una página nueva?).
- Estimar posibles riesgos (ej: "esto podría ralentizar la carga inicial").
